One way to build Cordova Android projects with CLI tools only

This is a guide to set up Cordova and an Android build environment to turn HTML5 projects into APK / AAB files locally, using command line tools only. The guide is for Windows (but can probably be adapted to Linux & macOS as well) and should work with any Cordova project; it's not specific to Construct 3 exports. This was written in February 2026 and might not age well.

Setting up the build environment

We need to install Node.js, OpenJDK, Android Command Line Tools and Gradle to build our Cordova Android project.

I like to use Scoop (an open source CLI installer for Windows). If you have the Extras bucket added, you can use this one liner to install all of the above:

scoop install nodejs-lts openjdk17 android-clt gradle-bin

The Cordova globally using the Node.js package manager:

npm install -g cordova

At this point we need a Cordova Android project. If you already have one, go to its directory and skip to the next section. If you're using Construct, go to Export, select Android and on the Cordova options screen select Cordova project.

If you just want to set up the build environment, you can create a blank Cordova Android project with the following commands:

cordova create blank-app
cd blank-app
cordova platform add android

Installing Android SDK Build Tools

At this point we can try to build the project. It will probably download some dependencies, but eventually fail and give you an error message.

cordova build android

The error message will tell you to install a specific version of the Android Build Tools (eg. 35.0.0), which may or may not be available from the Android SDK manager. It's best to list all available packages first:

sdkmanager --list

Look for "build-tools" and copy the line that matches or a tiny bit superseeds the version number requested by Gradle. Proceed to install the relevant version (35.0.0 in the example):

sdkmanager --install build-tools;35.0.0

If something goes wrong, you can uninstall a package like this:

sdkmanager --uninstall build-tools;35.0.0

You can check what's installed at the moment with this command:

sdkmanager --list

Installing more dependencies & first build

After Android SDK Build Tools are installed, we can attempt to build again:

cordova build android

This time it will hopefully proceed to install the remaining build dependencies and eventually finish the build successfully. If you have a slow Internet connection, it'll take a while. Subsequent builds will be faster.

The previous command creates a debug build. If you want to make a signed production build, you can use the following command as a template, just replace the ??? parts with the relevant values:

cordova build android --release -- --keystore="???.jks" --keystoreType jks --storePassword=??? --password=??? --alias=???

That's about it! Happy building :)
